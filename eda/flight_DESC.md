# Column Dictionary

209 rows (1 per flight) x 21 columns. Generated by `convert.ipynb` from the raw
timestep-level `drone.parquet` log.

| feature_name            | dtype   | unit     | description                                                                                                                                                                           |
| ----------------------- | ------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `flight`                | int64   | id       | Unique flight identifier.                                                                                                                                                             |
| `payload`               | float64 | grams    | Payload weight carried on the flight (constant per flight).                                                                                                                           |
| `route`                 | str     | category | Route/path code the drone flew (e.g. R1, A2, H).                                                                                                                                      |
| `altitude`              | str     | meters   | Target/flown altitude label for the flight (constant per flight; some flights show a multi-leg string like "25-50-100-25").                                                           |
| `date`                  | str     | date     | Calendar date the flight took place.                                                                                                                                                  |
| `time_day`              | str     | HH:MM    | Time of day the flight took off.                                                                                                                                                      |
| `duration_s`            | float64 | seconds  | Total flight duration (max of the timestep `time` column).                                                                                                                            |
| `total_distance_m`      | float64 | meters   | Total 3D path length flown, summed from step-to-step displacement in `position_x/y/z`. Proxy for trip "range".                                                                        |
| `wind_speed_mean`       | float64 | m/s      | Mean wind speed measured across the flight.                                                                                                                                           |
| `wind_speed_std`        | float64 | m/s      | Standard deviation of wind speed across the flight (wind variability/gustiness).                                                                                                      |
| `wind_x_mean`           | float64 | m/s      | Mean wind speed component along the x-axis (`wind_speed * sin(wind_angle)`).                                                                                                          |
| `wind_y_mean`           | float64 | m/s      | Mean wind speed component along the y-axis (`wind_speed * cos(wind_angle)`).                                                                                                          |
| `speed_mean`            | float64 | m/s      | Mean ground/air speed of the drone during the flight.                                                                                                                                 |
| `speed_max`             | float64 | m/s      | Maximum speed reached during the flight (proxy for the speed limit used).                                                                                                             |
| `velocity_mag_mean`     | float64 | m/s      | Mean of the 3D velocity vector magnitude (`‖velocity_x, velocity_y, velocity_z‖`).                                                                                                    |
| `velocity_mag_max`      | float64 | m/s      | Maximum 3D velocity magnitude reached during the flight.                                                                                                                              |
| `acceleration_mag_mean` | float64 | m/s²     | Mean of the 3D linear acceleration vector magnitude.                                                                                                                                  |
| `acceleration_mag_std`  | float64 | m/s²     | Standard deviation of the 3D acceleration magnitude (maneuvering intensity).                                                                                                          |
| `battery_voltage_mean`  | float64 | volts    | Mean battery voltage recorded during the flight.                                                                                                                                      |
| `battery_voltage_min`   | float64 | volts    | Minimum battery voltage recorded during the flight (closest point to depletion).                                                                                                      |
| `battery_needed`        | float64 | mAh      | **Target variable.** Total battery capacity consumed over the flight, computed as `sum(capacity_step)` where `capacity_step = (1/3.6) * dt * battery_current` (A·s converted to mAh). |

## Notes

- All columns except `flight`, `route`, `altitude`, `date`, `time_day` are derived by aggregating (mean/std/max/sum) the timestep-level telemetry within each flight.
- `payload`, `route`, and `altitude` are constant within a flight, so the `first` value is used.
- 3 flights (`211`, `212`, `213`) have a slightly negative `battery_needed` due to brief negative `battery_current` readings (sensor noise/regen) — flagged by the build script, not removed.
